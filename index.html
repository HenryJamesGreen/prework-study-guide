Git

  • GitHub Cheat Sheet
  • Repository (repo) - a folder for your project that contains all files and their version history.
  • Branch - GitHub workflow process - makes experience “clean”. Can merge changes back in to one place. Changes don't affect main branch or live site. Can make revisions without impacting anything else within repo.
  • Markdown file - a way to write and style text using plain text - easily converts to html (.md).
  • Commit - saving a new version of the file that you're working on. Adding, editing or deleting a file within your repo. Adding a change to branch you're working on.
  • Pull Request (‘PR’) - move the change that’s happening on your branch into the main branch. Asking for your changes to be pulled in or merged into another branch. Be specific about things you have changed etc. Within req it will list changes to any files you worked on. Next thing - someone reviews, discuss your work etc, have everyone agree upon changes etc to get published.
  • Merge(d) - when changes are actually pulled in from your branch to the main. In the process of being published and deployed into live site.
  • Issue - how we're tracking the work that is happening. Where you might report bug fixes or new features. Create an issue for any new content you want to add to site or fix. Tracked on content board (backing, to do, in progress). Include an attachment of word or google doc file so that everyone can see it. Can label the issue as to what it is (eg 'content').
